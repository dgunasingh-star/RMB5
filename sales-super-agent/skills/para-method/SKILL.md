# PARA+I — how to organize this workspace

Organize by ACTIONABILITY, not by topic (Tiago Forte's PARA). Four buckets plus Insights:

- projects/  — a specific outcome with a goal AND a foreseeable end/deadline; it finishes.
               Here: a live deal or opportunity ("IBM storage expansion — target close Q3").
- areas/     — an ongoing responsibility with a standard to maintain, no end date.
               Here: a key account you steward, your territory, your quota.
- resources/ — reference material and topics: playbooks, product specs, pricing, competitive intel.
- archive/   — anything above no longer active: closed/won/lost deals, dormant accounts.
- insights/  — short structured notes capturing what was learned when work completes.

PARA is a general method; here the folders happen to hold sales work, but the rules are the same.

## Filing rule
Ask in order: which active PROJECT does this serve? -> else which AREA? -> else a RESOURCE -> else
ARCHIVE. A project has a goal and an end; an area is ongoing. Break vague areas into concrete projects.

## When to create a project
- Quick one-off question -> just answer; no folder.
- A goal with an end/deadline (a deal to pursue) -> a PROJECT. If unsure or it looks ongoing, OFFER
  first; create projects/<deal>/ with brief.md only on yes. Never invent data.
- An ongoing account/territory to steward -> an AREA.

## Completion ritual (the "+I") — do BOTH, together
When a project reaches its end (won, lost, or closed):
1. Move it from projects/ to archive/.
2. At the same time, write an insight in insights/ — what happened, what worked, what didn't, the
   steps/tools used, any capability gap. Archiving without an insight loses the lesson.

## Multi-threading
For any project, identify at least two contacts on the buyer side (e.g. champion + economic buyer)
so the deal never hinges on a single relationship. Note them in the project's brief.md.

## Insight note format
Frontmatter: deal, outcome, date, synced: false. Body: 3-6 honest lines — what happened, what worked,
what to do differently next time.
